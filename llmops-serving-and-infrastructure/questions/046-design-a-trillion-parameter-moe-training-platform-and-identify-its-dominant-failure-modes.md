### Q: Design a trillion-parameter MoE training platform and identify its dominant failure modes.
* **Difficulty:** Principal
* **Category:** System Design
* **The 10-Second Pitch:** Compose expert/tensor/pipeline/data/context parallelism on topology-aware GPU pods, high-throughput data/checkpoint storage, router/expert telemetry, and fault-tolerant sharded state. All-to-all, imbalance, and recovery dominate.
* **The Deep Dive:** Size two different quantities: total parameters determine weight/optimizer/checkpoint memory and loading time; active parameters per token determine most forward/backward FLOPs. Build a topology-aware mesh combining expert, tensor, pipeline, data/FSDP, and possibly context parallelism. Keep frequent TP and expert all-to-all inside the fastest NVLink/NVSwitch/low-oversubscription fabric where possible; use PP for layer partitions and DP across larger failure/network domains. Router top-k dispatch packs tokens by destination expert, performs all-to-all, executes grouped GEMMs, and reverses the exchange. Capacity factor bounds worst-case buffers; auxiliary load-balancing/router-z losses and expert telemetry discourage collapse, but dropped-token policy and shared experts must be explicit.

  Balance pipeline stages using measured dense, routing, communication, and expert compute—not layer count. Overlap dispatch with local work without creating competing collectives on the same links. Shard dense parameters and optimizer state, checkpoint activations selectively, and accumulate enough tokens per expert for efficient GEMMs. Data ingestion records deterministic shard/sample position and mixture schedule. Checkpoints use per-rank temporary shards, checksums, a global manifest committed last, and logical tensor metadata so restart on a different mesh can reshard. Save model, optimizer, scaler, scheduler, RNG, router state/statistics, and data cursor.
* **Production Reality & Tradeoffs:** Dominant failures are expert hotspots/collapse, overflow or dropped tokens, tiny expert batches, all-to-all congestion, collective-tail amplification, pipeline imbalance, numerical divergence, silent data skew, hardware stragglers, and checkpoint storms/corruption. Monitor routing entropy, tokens/capacity/drops per expert and domain, per-expert gradient/update norms, all-to-all percentiles, stage bubbles, MFU, loss by data slice, ECC/link health, and restart integrity. Validate router/network behavior on topology-faithful pods before scaling; active-FLOP estimates alone radically understate network and state cost.
* **Red Flag:** Sizing only by active FLOPs and ignoring total weights/network.
