### Q: Explain macro, micro, weighted, per-class, multilabel, and threshold-dependent aggregation.
* **Difficulty:** Senior
* **Category:** Evaluation
* **The 10-Second Pitch:** Aggregation defines which examples or classes receive weight: macro treats classes equally, micro pools decisions, weighted macro follows support, and multilabel evaluation additionally chooses label-wise or example-wise reductions and thresholds.
* **The Deep Dive:** Compute each class one-vs-rest. Macro averages class metrics equally and exposes rare-class performance; weighted macro weights by class support and can resemble majority behavior. Micro sums TP/FP/FN across classes before computing the metric, so frequent classes dominate. Per-class values should accompany every aggregate because identical macro scores can hide different failures. In single-label multiclass classification, micro precision/recall/F1 collapse to accuracy; this is not generally true for multilabel data. Multilabel evaluation may average per label, per example, or globally, and includes subset accuracy, Hamming loss, and ranking metrics. Thresholds may be global, per-class, or budget-constrained; selecting them on the test set leaks labels. Calibrate/tune on validation data against an explicit utility such as $benefit*TP-cost_fp*FP-cost_fn*FN$, possibly under recall, capacity, or fairness constraints.
* **Production Reality & Tradeoffs:** Weighted and micro metrics are stable but can erase rare critical classes; macro is noisy when support is tiny. Production label prevalence and annotation missingness may differ from the test set. Publish supports, uncertainty, thresholds, and slice confusion matrices.
* **Red Flag:** Saying macro is always fair or micro is always accurate without explaining what each reduction weights.
