### Q: Design retrieval triggering and corrective re-retrieval: when to retrieve at all, self-assessment, and Self-RAG/CRAG-style loops.
* **Difficulty:** Senior
* **Category:** RAG
* **The 10-Second Pitch:** Gate retrieval with a retrieve-or-not decision, grade what came back before generation, and on a bad grade take one bounded corrective action—rewrite, widen, or fall back to web search—rather than either always retrieving or trusting the first candidate set.
* **The Deep Dive:** The trigger is a classification problem: does answering require corpus evidence? Useful signals are query type (chit-chat and formatting requests never need retrieval), temporal markers, rare or tenant-specific entities, and whether the question targets policy, pricing, or documents that only exist in the index. Implementations range from a distilled classifier over a few thousand labeled queries, to Self-RAG's approach of training the generator to emit a special retrieve token mid-generation, so retrieval can also trigger lazily when the model reaches a claim it cannot support. The trap is overconfident skipping: the model "knows" an answer that the corpus has since superseded—last year's leave policy, a deprecated API default—so the gate must be biased toward retrieving whenever freshness or tenant specificity is plausible, and the skip path must be audited, not assumed.

  Self-assessment grades the retrieved set before it reaches the generator. CRAG frames this as a retrieval evaluator emitting correct, ambiguous, or incorrect; in practice a calibrated cross-encoder or reranker score works: if the top span's score clears a threshold validated on labeled data, proceed; in a gray band, act; below a floor, treat retrieval as failed. Corrective actions form an escalation ladder, each strictly widening: rewrite the query (decontextualize pronouns, add entities from conversation state), widen the search (raise $k$, drop metadata filters, shift hybrid weighting toward lexical for rare terms), decompose into sub-queries, then fall back to web search where policy allows, and finally abstain with an explicit "not found in your documents." Each rung is one bounded step—at most two corrections per query—because the loop otherwise doubles or triples p95 latency for exactly the queries that were already failing.

  Two failure modes dominate. First, correlated self-assessment: if the same model both grades evidence and generates the answer, its blind spots align, and it passes plausible-but-wrong evidence it will then happily cite; an independent scorer (a reranker trained on relevance labels, not the generator) breaks the correlation. Second, threshold drift: reranker score distributions shift when the corpus or embedder changes, so thresholds must be recalibrated against a fixed labeled set on every model or index rollout. The falsifiable test targets the skip gate directly: sample $200$ queries the gate skipped, force-retrieve for all of them, and have judges label how many answers materially change when evidence is supplied. If more than about $5$ percent change, the gate is miscalibrated and its threshold must move toward retrieval; rerun this audit monthly, because skip-rate that looks stable can hide a growing stale-knowledge segment.
* **Production Reality & Tradeoffs:** The gate saves real money—if 30 percent of traffic skips retrieval, you drop that fraction of embedding, ANN, and rerank cost plus roughly $1\,000$ to $2\,000$ context tokens per query—but the corrective path spends it back on the hard tail, so track the correction-rate time series as a first-class drift signal: a rising rewrite rate usually means query drift or index staleness, not a broken evaluator. Cache evaluator verdicts keyed to query and index version, log every rung taken for replay, and never let web fallback bypass tenant data-egress policy.
* **Red Flag:** Using the generator's own confidence in its final answer as the retrieval trigger, or building an unbounded correct-and-retry loop where the model re-grades its own retrievals until it "succeeds," which converges on confident hallucination at maximum latency.
