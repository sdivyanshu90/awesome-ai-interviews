### Q: How do reasoning models change agent design: interleaved thinking, thinking budgets, and when ReAct scaffolding is obsolete.
* **Difficulty:** Senior
* **Category:** Architecture
* **The 10-Second Pitch:** ReAct existed because base models would act without deliberating, so the harness forced a written "Thought:" before every action. Reasoning models internalize that step: they emit a native, RL-trained thinking phase, and with interleaved thinking they deliberate between tool calls inside one assistant turn — think, call, read the result, think again. The scaffold's thought field becomes redundant; the harness instead controls the thinking budget per step and must correctly preserve or strip thinking blocks across turns, which has cache and behavior consequences. Explicit scaffolds survive where durability, auditability, or weak models demand them.
* **The Deep Dive:** In classic ReAct the reasoning-action alternation is a prompt-engineering artifact: the model is instructed to emit Thought / Action / Observation lines, and the harness parses them. A reasoning model produces the same alternation natively — a thinking segment the API returns as a distinct block (Anthropic's thinking blocks, OpenAI's reasoning items), followed by tool calls. Interleaved thinking closes the remaining gap: instead of thinking only once before the first call, the model can reflect on each tool result before choosing the next action, which is precisely the ReAct loop, executed inside the model rather than by the harness. Layering a hand-rolled Thought: instruction on top of this duplicates deliberation, spends tokens twice, and can degrade behavior, because the model was post-trained to structure its reasoning its own way — the scaffold now fights the training rather than compensating for its absence.

  Deliberation became a metered, per-call resource. Anthropic exposes a token budget for thinking, OpenAI a reasoning-effort level, Gemini a thinking budget, so a harness can spend where decisions are hard and starve where they are mechanical: triaging which failing test to attack merits a large budget, emitting the file-read call that the plan already determined merits almost none. This changes agent economics — latency and cost now scale with deliberation, not only with context — so mature harnesses classify step types and assign budgets the way they already assign tool permissions. A falsifiable experiment: run a fixed task suite at low and high budgets; if success does not improve with budget, you are paying for deliberation your task distribution does not need.

  Thinking is also state the harness must shepherd. Providers treat prior-turn thinking as disposable — Anthropic strips thinking blocks from previous assistant turns automatically — but within an active tool-use turn the current thinking block must be returned unmodified with the tool result (blocks are cryptographically signed; tampering is rejected), and OpenAI's Responses API similarly passes reasoning items back so the chain survives across calls. Failure trace: a harness that rebuilds conversation history by hand drops the signed thinking block when appending a tool result; the API rejects the request — or, in a laxer integration, the model continues without its interrupted plan, silently re-derives context from visible messages, and picks a different action than the one it was mid-way through justifying. Cache interacts the same way: prompt caching depends on stable prefixes, so history edits and configuration flips (toggling thinking on or off mid-session) invalidate cached segments and multiply input cost.

  Scaffolding is not dead everywhere. Ephemeral thinking is lost on compaction, so durable plan artifacts — an explicit todo list or plan file the agent maintains — still outperform trusting the model to re-think its plan every turn. External checkpoints (approval gates, budget checks) must live in the harness regardless of where thinking happens. And small or non-reasoning models still benefit from classic ReAct prompting. The honest test: A/B your ReAct scaffold against native thinking on the same tasks and tools; retire whichever loses.
* **Production Reality & Tradeoffs:** Thinking tokens are billed as output and dominate cost on deliberation-heavy steps, so budget tuning is a real line item. Some providers return summarized rather than raw thinking, which weakens audit trails — log what you actually received, not what you assume the model thought. Interleaved thinking remains provider-specific in its API contract, so harnesses need per-provider adapters, and regression-test them on provider upgrades.
* **Red Flag:** Bolting a Thought/Action/Observation prompt onto a reasoning model "because agents need ReAct", or hand-editing assistant turns containing signed thinking blocks. The loop now lives partly inside the model; the harness's job is budget, state custody, and durability — not narrating the reasoning.
